New Website Design

Well... we go back in time even more. Borders? YES, Font? PIXELS, Readability? Actually a little better, CSS-Code? More jank. The new design has inspiration from different directions. From Serial Experiments Lain, to my own game to Neon Genesis Evangelion and old Linux Desktops like 4Dwm themes or old OpenBox setups or very old GNOME. I think you get it.

I needed to fix many (too many) small things on many (too many) old pages because they had broken HTML part's. Nothing mayor but small things that stick out a lot because of the way the new CSS code works with different parts of the pages.
I surprisingly did not need to use any janky HTML code to make this site look like this, if anything the code is now even more HTML specification compliant the ever before, just very clever usage of CSS and some tutorials.

Will the design change again soon: No. It is a lot of trying and tinkering to change a full websites design, make it coherent and good and don't find many old typing errors that need fixing (HTML is way to lenient with coding mistakes and syntax). That is work that you don't want to do often. I will change it when the old design gets/feels stale again. The nice part is that it forces me to cleaning the HTML code more and more because I ONLY want to use CSS tho change the design. As always I love making everything absurdly modular. Even RE//POSSESSED's code is very (as much as game engines and game architecture allows) modular and reusable. That, for example, gives my inventory system in the game the theoretical absurdity of picking up anything that can collide with the pickup Raycast... theoretically.
I did this redesign the last two weeks of and on when I needed a break from game-dev.
